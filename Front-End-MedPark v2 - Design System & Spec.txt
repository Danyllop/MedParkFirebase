1. STACK TECNOLÓGICA DE ALTA PERFORMANCE
--------------------------------------------------------------------------------
- Framework: React.js (Gestão de estados complexos e real-time).
- Styling: Tailwind CSS (Consistência e produtividade).
- UI Components: Radix UI / Shadcn UI (Acessibilidade e customização total).
- Ícones: Lucide React (Minimalistas) e Phosphor Icons (Específicos para veículos).
- Animações: Framer Motion (Transições de menu e modais).
- Comunicação: Socket.io-client (Atualização de vagas via WebSockets).

2. IDENTIDADE VISUAL (DESIGN TOKENS)
--------------------------------------------------------------------------------
- Paleta de Cores (Cromatismo):
    * Background Primário: #0f172a (Deep Navy - Fundo e Inputs).
    * Background Secundário: #1e293b (Slate 800 - Cards e Sidebar).
    * Accent Color: #3b82f6 (Vibrant Blue - Ações e Foco).
    * Texto Primário: #f8fafc (Branco Gelo).
    * Texto Secundário: #94a3b8 (Cinza Azulado).
- Estados Visuais:
    * Sucesso (Vaga Livre): #22c55e.
    * Aviso (Vaga Reservada): #eab308.
    * Erro (Bloqueado/Ocupada): #ef4444.
- Tipografia: Inter ou Geist Sans (14px base para tabelas).
- Raio de Borda (Radius): 0.75rem (12px) p/ Cards; 0.5rem (8px) p/ Botões/Inputs.

3. ESPECIFICAÇÃO DE TELAS OPERACIONAIS
--------------------------------------------------------------------------------

3.1. TELA DE LOGIN & PRIMEIRO ACESSO
  - Botão de Ação: Texto "Entrar".
  - Senha: Input com ícone de olho (Toggle visibility).
  - Lógica: Se detectar senha padrão "Mud@1234", redireciona para página 
    de troca obrigatória antes de liberar o painel.

3.2. SIDEBAR RETRÁTIL (NAVEGAÇÃO)
  - Itens: Dashboard, Funcionários, Prestadores, Portaria A, Portaria E, 
    Gestão de Usuários, Infrações, Relatórios.
  - Ativo: Background azul suave + barra lateral de 3px em Vibrant Blue.

3.3. PORTARIA A (90 VAGAS - DIRETORIA)
  - Busca Universal: Campo único (Nome, CPF, Placa, Adesivo).
  - Regra Especial: Vagas Diretoria bloqueadas visualmente Seg-Sex (06h-18h).
  - Mapa: Grid interativo refletindo status em tempo real via WebSocket.

3.4. PORTARIA E (200 VAGAS - ALMOXARIFADO)
  - Busca Universal: Integrada com tabelas de Funcionários e Prestadores.
  - Fluxo: Campo extra de "Destino" com destaque para "Almoxarifado".

4. REGRAS DE FORMULÁRIO E INTEGRIDADE
--------------------------------------------------------------------------------
- Cadastro de Funcionários: Validação de CPF e Placa ÚNICOS.
- Cadastro de Prestadores: Permite múltiplos registros por CPF/Placa (Flexível).
- Campo Adesivo (Funcionários): 
    * Input manual para os 10.000 registros antigos.
    * Botão [+] gerador: Automático (Max < 11000 ? 11000 : Max + 1).
- Feedback: Notificações Toast no canto inferior direito para Sucesso/Erro.

5. ACESSIBILIDADE E UX
--------------------------------------------------------------------------------
- Dark Mode Nativo: Focado na redução de fadiga para turnos de 24h.
- Responsividade: Otimizado para monitores desktop de portaria (15" a 24").
- Performance: Uso de filtragem Client-side para tabelas rápidas.
